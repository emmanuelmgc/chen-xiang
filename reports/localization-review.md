## 3. 日期、数字、货币格式检查

### ⚠️ 发现的问题

#### 3.1 货币格式

**问题**: 阿拉伯语版本中货币格式不正确

**当前格式** (在 `products.ts` 和 `gift_sets.ts` 中):
```
priceRange_ar: "99 - 179 SAR"
priceRange_en: "99 - 179 SAR"
```

**问题**:
1. 阿拉伯语应使用阿拉伯-印度数字 (٠١٢٣٤٥٦٧٨٩) 而非西阿拉伯数字 (0123456789)
2. 阿拉伯语中 "SAR" 应放在数字前面或改为阿拉伯语 "ريال"

**建议格式**:
```
阿拉伯语: "٩٩ - ١٧٩ ريال" 或 "SAR ٩٩ - ١٧٩"
英语: "99 - 179 SAR"
```

#### 3.2 数字格式

**文件**: `src/data/products.ts`

当前所有数字使用西阿拉伯数字 (如 `30 عصا / 15 جم`)，阿拉伯语版本应使用阿拉伯-印度数字 (`٣٠ عصا / ١٥ جم`)。

#### 3.3 日期格式

未在审查的文件中发现明确的日期显示格式。如有日期显示功能，应确保：
- 阿拉伯语: DD/MM/YYYY (如 `٠٢/٠٦/٢٠٢٦`)
- 英语: MM/DD/YYYY (如 `06/02/2026`)

## 5. 修复实施

### 5.1 修复货币和数字格式

由于修改数据文件会影响整个网站，建议创建一个格式化工具函数：

**建议创建**: `src/lib/formatters.ts`

```typescript
// 格式化数字为阿拉伯-印度数字
export function toArabicNumerals(num: string | number): string {
  const arabicDigits = ['٠', '١', '٢', '٣', '٤', '٥', '٦', '٧', '٨', '٩'];
  return String(num).replace(/[0-9]/g, (digit) => arabicDigits[parseInt(digit)]);
}

// 格式化价格范围
export function formatPriceRange(priceRange: string, locale: 'ar' | 'en'): string {
  if (locale === 'ar') {
    // 转换为阿拉伯-印度数字
    return toArabicNumerals(priceRange) + ' ريال';
  }
  return priceRange;
}
```
